# Finance Tracker – Notion‑style Local Web App

## Overview
A premium‑looking, **Notion‑style** personal finance dashboard that runs entirely in the browser. It includes:
- Sidebar navigation with dark‑mode toggle
- Dashboard with summary cards, a seasonal table and interactive charts (net‑worth, TT/Week, VA/Week)
- Editable weekly data table with CSV import/export
- Accounts view with historical balances and a distribution chart
- Projections tool to simulate future growth
- Settings page (theme, currency, data reset)
- Import/Export of the whole data set (JSON/CSV)

All data is persisted in **localStorage**, so your changes survive page reloads.

## Getting Started
1. **Open the app**
   - Simply double‑click `index.html` in the `finance-tracker` folder, or open it with your browser via `file:///c:/antigrav/finance-tracker/index.html`.
   - The app works offline – no server is required.

2. **Optional: Run a local dev server** (useful for Chrome security policies)
   ```powershell
   # PowerShell (Windows) – Python 3 must be installed
   cd C:\antigrav\finance-tracker
   python -m http.server 8080
   ```
   Then navigate to `http://localhost:8080`.

3. **Explore the UI**
   - Use the sidebar to switch between Dashboard, Data Table, Accounts, Projections, Import/Export, and Settings.
   - Click the moon/sun icon or the toggle in Settings to switch between light and dark themes.
   - Edit any cell in the tables directly – changes are saved automatically.
   - Add new rows via the **Add** buttons; a modal will guide you.
   - Export your weekly data as CSV or the whole dataset as JSON.
   - Import a previously exported JSON file by dragging it onto the drop zone or using the file picker.

## Customisation
- **Currency** – Change the display currency in Settings (EUR, USD, GBP). The formatter updates instantly.
- **Initial data** – The default data is generated in `app.js` (functions `getDefaultSeasonData`, `getDefaultWeeklyData`, `getDefaultAccountData`). Edit those functions if you want a different starting point.
- **Styling** – The design system lives in `style.css`. Adjust CSS variables under `:root` for colours, spacing, or fonts.

## Known Limitations
- The app relies on **Chart.js** from a CDN; an internet connection is required for the charts.
- Import/export works with the JSON schema generated by the app. Importing other formats may cause errors.
- When opening the file via `file://` some browsers (e.g., Chrome) may block certain features; running a simple local server as shown above avoids this.
